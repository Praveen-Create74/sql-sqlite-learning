# SQL Learning Repository

This repository documents my journey through SQL fundamentals, based on Scrimba’s structured modules. It includes practice queries, screenshots, and notes that showcase my progress from beginner concepts to advanced logic.

---

## 📚 Concepts Covered

### 🔹 Phase 1 – Beginner SQL
- **Introduction to Databases**  
  - Tables, rows, and columns  
  - Primary keys and uniqueness  
- **Basic Queries**  
  - `SELECT` for retrieving data  
  - Filtering with `WHERE`  
  - Sorting results using `ORDER BY`  
  - Limiting output with `LIMIT`  
- **Aggregate Functions**  
  - `COUNT()`, `SUM()`, `AVG()`, `MIN()`, `MAX()`  
  - Grouping with `GROUP BY`  
  - Filtering groups with `HAVING`  
- **Practice Focus**  
  - Simple queries on single tables  
  - Understanding how SQL retrieves and processes data  

---

### 🔹 Phase 2 – Joining Tables & CRUD Operations
- **CRUD Basics**  
  - `INSERT` → add new records  
  - `UPDATE` → modify existing records  
  - `DELETE` → remove records  
- **Joins**  
  - `INNER JOIN` → combine matching rows across tables  
  - `LEFT JOIN` → include unmatched rows from the left table  
  - `RIGHT JOIN` / `FULL JOIN` (conceptual understanding)  
- **Relationships**  
  - Foreign keys and linking tables  
  - Practical examples: students ↔ courses, orders ↔ customers  
- **Practice Focus**  
  - Writing queries across multiple tables  
  - Performing CRUD operations with conditions  

---

### 🔹 Phase 3 – Advanced SQL Logic
- **Subqueries**  
  - Nested `SELECT` statements  
  - Using subqueries in `WHERE` and `FROM` clauses  
- **Aliases & Readability**  
  - `AS` for renaming columns and tables  
- **Distinct & Filtering**  
  - `DISTINCT` to remove duplicates  
  - Pattern matching with `LIKE` and wildcards (`%`, `_`)  
- **Conditional Logic**  
  - `CASE` statements for custom outputs  
- **Constraints & Integrity**  
  - `NOT NULL`, `UNIQUE`, `CHECK`  
  - Ensuring data validity  
- **Indexes & Performance**  
  - Speeding up queries with indexes  
- **Transactions**  
  - `BEGIN`, `COMMIT`, `ROLLBACK` for safe operations  
- **Practice Focus**  
  - Complex queries combining multiple concepts  
  - Building logic for real-world scenarios
